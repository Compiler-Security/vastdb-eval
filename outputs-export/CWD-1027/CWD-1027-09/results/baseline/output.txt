

## Root Cause

The vulnerability is a classic memory leak (CWE-401). At line 342, `avio_open_dyn_buf(&dyn_bc)` allocates a dynamic buffer and stores the `AVIOContext` in `dyn_bc`. The buffer should be released via `avio_close_dyn_buf` (as done in `put_packet` at line 314), but when `nut->sp_count` equals 0, the condition at line 343 evaluates to false, causing the code to skip `write_index` and `put_packet` entirely and return directly at line 349. This leaves the dynamically allocated buffer (the `DynBuffer` struct and its `io_buffer`) never released, causing a memory leak.
