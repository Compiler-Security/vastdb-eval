## Root Cause

The vulnerability is a missing release of memory after effective lifetime (CWE-401). In `glutAddSubMenu` function at line 276 in `bad.c`, a `menuEntry` structure is allocated via `calloc(sizeof(SFG_MenuEntry), 1)`. If `fgState.ActiveMenus` is non-zero at line 280, the function calls `fgError` at line 281, which terminates the program via `exit(1)` at line 266 in the `fgError` function without freeing the allocated `menuEntry` memory. The program exits abruptly, leaving the heap-allocated memory unfreed.

## Trigger Path

Since this is a direct API entry point (not a chain of callables), there is only one stack frame from external entry to vulnerable operation:

1. `bad.c:270` `glutAddSubMenu`
   - External API entry point that takes menu label and submenu ID as arguments
   - At line 276, allocates `menuEntry` with `calloc(sizeof(SFG_MenuEntry), 1)` - the source/memory allocation
   - At line 280, checks `if (fgState.ActiveMenus)` - condition that triggers error path
   - At line 281, calls `fgError("Menu manipulation not allowed while menus in use.")` - the sink
   - `fgError` (lines 237-266) prints error and calls `exit(1)` at line 266, terminating without freeing `menuEntry`
