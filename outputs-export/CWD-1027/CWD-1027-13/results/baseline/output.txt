

## Root Cause

The vulnerability is a memory leak where `menuEntry` (allocated via `calloc` at line 276) is never freed before program termination. When `fgState.ActiveMenus` is non-zero (indicating menus are in use), line 281 calls `fgError()` which invokes `fgDeinitialize()` and then `exit(1)` at line 266. This terminates the program without freeing the `menuEntry` memory allocated at line 276. The allocated `menuEntry` pointer becomes unreachable immediately after the `fgError()` call, causing a classic missing memory release (CWE-401).

## Trigger Path

1. `bad.c:270` `glutAddSubMenu`
   - External/top-level API entry. Receives label and subMenuID from caller. Allocates `menuEntry` at line 276 via `calloc`.

2. `bad.c:280` condition check
   - Checks `if (fgState.ActiveMenus)` - if true, control flows to `fgError()` at line 281.

3. `bad.c:281` `fgError("Menu manipulation not allowed while menus in use.");`
   - Sink location. Triggered when menus are in use. Calls `fgError()` which terminates via `fgDeinitialize()` and `exit(1)` without freeing the allocated `menuEntry`.
