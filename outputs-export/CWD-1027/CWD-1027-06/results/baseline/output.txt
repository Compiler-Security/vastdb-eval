

## Root Cause

The vulnerability is a **missing release of heap-allocated memory** (CWE-401). At line 353, the function `vsmprintf()` allocates a buffer via `malloc()` and stores the pointer in `buf`. When `screen_display` is NULL (line 354), the function returns NULL at line 357 without freeing this allocated buffer, causing a memory leak. The allocated memory becomes unsafe because the program loses the only reference to it, preventing any deallocation.

## Trigger Path

1. `bad.c:353` `vsmprintf(question, ap)`
   - Allocates a heap buffer and assigns pointer to `buf`

2. `bad.c:357` `return( NULL );`
   - Returns NULL when `screen_display==NULL`, never freeing `buf`
