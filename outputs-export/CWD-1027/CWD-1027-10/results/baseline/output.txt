## Root Cause

The vulnerability is a **Missing Release of Memory after Effective Lifetime** (CWE-401). In the function `streamGetEdgeID()` at lines 419-431, a `streamIterator` structure `si` is declared as a local variable on the stack. Inside `streamIteratorStart()` (the vulnerable location at line 423), the iterator initializes its internal `raxStack` and calls `raxSeek()` which can trigger dynamic memory allocation in the raxStack (when pushing more than 32 items). However, `streamGetEdgeID()` returns at line 430 **without calling `streamIteratorStop()`** to free any dynamically allocated memory held by the iterator's internal stack. This causes a memory leak because the heap-allocated stack buffer is never freed.

## Trigger Path

1. `bad.c:419` `streamGetEdgeID(stream *s, int first, int skip_tombstones, streamID *edge_id)`
   - Declares local `streamIterator si` variable; calls `streamIteratorStart()` with the iterator address
   
2. `bad.c:376` `streamIteratorStart(streamIterator *si, stream *s, streamID *start, streamID *end, int rev)`
   - Initializes iterator internal state; calls `raxStart()` on `&si->ri` to set up internal raxIterator
   
3. `bad.c:358` `raxStart(raxIterator *it, rax *rt)`
   - Calls `raxStackInit(&it->stack)` to initialize the rax stack; calls `raxSeek()` which may push >32 items
   
4. `bad.c:430` `return;`
   - **SINK**: Returns without calling `streamIteratorStop()` to free dynamically allocated raxStack memory, causing the memory leak.
