## Root Cause

The `png_struct` allocated by `png_create_read_struct()` at line 582 in `read_png()` is never freed. When the function returns after an error (due to failed allocation logged via `display_log()`), the allocated memory is abandoned. Additionally, when `update_display()` moves the structure to `dp->original_pp` at line 635, the original `dp->read_pp` is set to NULL without freeing the memory first. The `display_destroy()` at line 994 is called too late (after the return 99), and there's no call to `png_destroy_read_struct()` anywhere in the code to properly release these allocations.
