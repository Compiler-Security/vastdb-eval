## Root Cause

The vulnerability is CWE-401: Missing Release of Memory after Effective Lifetime. At line 582 in `read_png()`, `png_create_read_struct()` allocates a png_struct and stores it in `dp->read_pp`. When an error condition occurs in `do_test()` (line 964: `if (ret > QUIET) return 99;`), the function returns 99 immediately without freeing the memory allocated to `dp->read_pp`. The memory is only freed in `display_destroy(&d)` which is called after the main loop completes (line 994), but never in the early error return path.

## Trigger Path

1. `bad.c:867` `main`
   - Entry point; loops through command-line files calling `do_test(&d, argv[i])`

2. `bad.c:850` `do_test`
   - Calls `test_one_file(dp, file)` at line 857; if an error causes `ret > QUIET` (line 961-964), returns 99 without freeing memory

3. `bad.c:698` `test_one_file`
   - Calls `update_display(dp)` at line 706 to cache the file and read original PNG data

4. `bad.c:623` `update_display`
   - Calls `read_png(dp, &dp->original_file, "original read", 0)` at line 632

5. `bad.c:565` `read_png`
   - At line 582, allocates png_struct via `png_create_read_struct()` stored in `dp->read_pp`; if error occurs during subsequent operations, memory leaks when `do_test()` returns 99 at line 964
