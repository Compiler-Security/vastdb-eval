## Root Cause

The vulnerability is a memory leak in `ff_frame_thread_encoder_init` (lines 1426-1541). At line 1506, a new `AVCodecContext *thread_avctx` is allocated via `avcodec_alloc_context3(avctx->codec)`. If any subsequent operation fails (lines 1507, 1512, 1517, or 1524), the code jumps to the `fail` label (line 1536) which calls `ff_frame_thread_encoder_free(avctx)` and returns -1 at line 1540. However, the `thread_avctx` allocated at line 1506 is never freed before returning -1, causing a memory leak. The known vulnerable location (source at line 1506, sink at return -1) triggers this CWE because the early allocation creates a context that is abandoned if any error occurs afterward.

## Trigger Path

1. `bad.c:1506` `ff_frame_thread_encoder_init` - Allocates `thread_avctx` via `avcodec_alloc_context3(avctx->codec)`, which allocates heap memory for a thread-specific codec context.
2. `bad.c:1536` `fail` label - Jumped to when any subsequent operation fails (lines 1507, 1512, 1517, or 1524), indicating an error condition.
3. `bad.c:1540` return -1 - Returns error code without freeing the `thread_avctx` allocated at line 1506, causing memory leak.
