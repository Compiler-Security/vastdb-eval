## Root Cause

At line 273, a `SFG_MenuEntry` structure is allocated via `calloc()`. If `fgState.ActiveMenus` is non-zero (checked at line 276), the code calls `fgError()` at line 277 which eventually invokes `exit(1)` (line 265). The allocated `menuEntry` pointer is never freed before the program exits, causing a memory leak. This is the classic CWE-401 pattern: memory is allocated but lost when an error path terminates the program without proper cleanup.
