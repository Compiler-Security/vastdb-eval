

## Root Cause

The vulnerability is a **missing release of memory** (CWE-401). In `glutAddMenuEntry`, a `menuEntry` structure is allocated via `calloc()` at line 273. Immediately after allocation (line 276), the code checks if `fgState.ActiveMenus` is truthy. If so, it calls `fgError()` which may terminate the program. When termination occurs, the allocated memory at line 273 is never freed because the code lines that would use or free it (lines 279-283) are skipped. Thus the memory becomes unreachable/leaked after the effective lifetime ends.

## Trigger Path

1. `bad.c:269` `glutAddMenuEntry`
   - Allocates `menuEntry` via `calloc()` at line 273, then checks `fgState.ActiveMenus` at line 276 which triggers `fgError()` at line 277 if active menus exist, causing early exit that bypasses cleanup code.
