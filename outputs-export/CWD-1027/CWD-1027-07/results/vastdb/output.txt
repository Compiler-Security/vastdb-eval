## Root Cause

In `bad.c`, the function `param_parse` (line 371) allocates memory at line 407 using `av_iamf_param_definition_alloc` to create an `AVIAMFParamDefinition` structure. At line 447-449, when a validation check fails (`!mode && !constant_subblock_duration && total_duration != duration`), the function returns `AVERROR_INVALIDDATA` at line 449 to indicate an error. However, unlike other error return paths in the same function (lines 462, 476 which properly call `av_free(param)`), this particular return path does NOT free the allocated `param` memory. This causes a memory leak (CWE-401 Missing Release of Memory after Effective Lifetime).

## Trigger Path

I could not prove a full caller stack from the code. The function `param_parse` is a static function defined at line 371 but has no internal callers in this file. The vulnerability triggers from external callers that invoke `param_parse`, which then hits the vulnerable code path at line 449.

1. `bad.c:407` `av_iamf_param_definition_alloc`
   - Allocates memory for `AVIAMFParamDefinition` structure and assigns to `param`

2. `bad.c:449` `return AVERROR_INVALIDDATA` (vulnerable sink)
   - Returns error code without calling `av_free(param)`, causing memory leak
