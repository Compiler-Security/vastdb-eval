## Root Cause

The vulnerability is a **memory leak (CWE-401)** in the `param_parse` function at lines 447-450 in `bad/bad.c`. When the condition `!mode && !constant_subblock_duration && total_duration != duration` is triggered (indicating invalid subblock durations), the function returns `AVERROR_INVALIDDATA` without freeing the `param` memory that was allocated at line 407 via `av_iamf_param_definition_alloc()`. The allocated `param` buffer is lost when this return path is taken, causing a memory leak. The good version fixes this by adding `av_free(param)` before returning at line 449.

## Trigger Path

This test case appears to be a self-contained fragment without direct external callers in the file. Based on the similar full test case in CWD-1031, the trigger path would be:
