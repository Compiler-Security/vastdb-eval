## Root Cause

The vulnerability is an out-of-bounds write in the `OPENSSL_uni2utf8` function. The `asclen` variable is used both as a buffer offset (line 169: `bmp_to_utf8(asctmp + asclen, uni + i, unilen - i)`) and is incremented by the return value `j` from `bmp_to_utf8` (lines 157 and 174: `asclen += j`). The `asclen` value is calculated in a first loop (lines 144-158), then memory is allocated, and a second loop (lines 168-175) performs the actual conversion using `asclen` as an offset. When malformed UTF-16 input causes `bmp_to_utf8` to return unexpected values or when the first loop calculation differs from actual bytes written in the second loop, the offset exceeds the allocated buffer size, causing an out-of-bounds write at line 179: `asctmp[asclen] = '\0'`.
