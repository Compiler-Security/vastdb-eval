

## Root Cause

The vulnerability is an out-of-bounds write in the `asn1_type_get_int_oct` function at line 199 when it performs `memcpy(data, ASN1_STRING_get0_data(oct), n)`. The function calculates `n` as the minimum of either the actual ASN1_OCTET_STRING length (`ret`) or the provided `max_len`. However, the caller `evp_cipher_get_asn1_aead_params` passes `i` (the actual octet string length from a prior call) as `max_len` without ensuring it fits in the destination buffer `iv` (which is only `EVP_MAX_IV_LENGTH` = 16 bytes). When the ASN1_OCTET_STRING data exceeds 16 bytes, `n` equals that larger size, causing memcpy to write beyond the 16-byte `iv` buffer—triggering CWE-787.
