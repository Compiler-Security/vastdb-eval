## Root Cause

The vulnerability is a CWE-787 (Out-of-bounds Write) caused by a mismatch between the actual number of planes in a video frame and the code's assumption of 4 planes. In `config_input` (line 555), the code unconditionally sets `planeheight[0]` through `planeheight[3]` for all 4 possible planes without checking `s->nb_planes`, which is set to `av_pix_fmt_count_planes(inlink->format)`. Certain pixel formats (e.g., grayscale with 1 plane, YUV422 with 3 planes) have fewer planes than 4, but the code still writes to indices beyond the valid plane count. When `deinterlace_slice` processes frames, it uses `s->planeheight[plane]` where `plane` comes from the loop in `filter()` that iterates based on `s->nb_planes`, leading to out-of-bounds access on the `planeheight` array.
