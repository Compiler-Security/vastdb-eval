## Root Cause

The vulnerability is an **out-of-bounds write** in the `gaussian_blur` function at line 216: `dst[1] = src[1]`. The root cause is:
1. In `config_props` (line 193), `plane->width` is calculated as `AV_CEIL_RSHIFT(inlink->w, hsub)` where `hsub` is the horizontal chroma subsampling factor (typically 1 for YUV420p chroma planes).
2. For small input widths (e.g., width=1), after the right-shift by `hsub=1`, `plane->width` becomes 0 or 1.
3. In `filter_frame` (line 287), `gaussian_blur` is called with `width` parameter set to `plane->width`.
