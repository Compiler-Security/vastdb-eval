## Root Cause

The vulnerability is a CWE-415 Double Free in the `chains` global pointer. At line 511 in function `get_num_dynamic_syms`, after calling `free(chains)`, the code incorrectly assigns `buckets = NULL` instead of `chains = NULL`. This leaves the `chains` pointer with a dangling reference to freed memory. Later, when `process_symbol_table` is called, it attempts to `free(chains)` again at lines 1606 and 1681, causing a double free.

## Trigger Path

1. `bad.c:1278` `process_object`
   - Entry function that initializes and processes ELF data structures.

2. `bad.c:1335` calls `process_dynamic_section` (line 678)
   - Processes dynamic section and calls helper functions.

3. `bad.c:741` calls `get_num_dynamic_syms` (line 460)
   - Frees `chains` at line 511 but incorrectly nullifies `buckets` instead of `chains`, leaving a dangling pointer.

4. `process_symbol_table` (line 335)
   - Called from `process_object`, attempts to free `chains` at lines 1606 and 1681, causing double free on the already-freed `chains` pointer from step 3.
