

## Root Cause

The vulnerability is a CWE-415 Double Free in the window tagstack cleanup. When a buffer is wiped via `close_buffer()`, it calls `mark_forget_file()` which iterates through the window's tagstack and calls `tagstack_clear_entry()` to free each tagname entry. However, this frees the memory and sets the **local** variable in the function to NULL, but does NOT update or clear the entries in the window's actual `w_tagstack` array. When the window is subsequently freed (e.g., via `win_free_popup()` -> `win_free()`), the code at lines 620-624 iterates based on `w_tagstacklen` and calls `vim_free(wp->w_tagstack[i].tagname)` on the same pointers that were already freed, causing a double free.

## Trigger Path

Unable to determine complete call stack from external/top-level entry. The available code shows:

3. `bad.c:553` `win_free`
   - Iterates from 0 to w_tagstacklen and calls vim_free(wp->w_tagstack[i].tagname) - the sink location (line 622)
