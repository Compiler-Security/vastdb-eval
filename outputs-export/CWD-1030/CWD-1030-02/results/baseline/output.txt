## Root Cause

The vulnerability stems from an inconsistent state between the pre-allocated `all_relocations_root` array and the actual relocation data. In `process_relocs()` at line 1438, `update_all_relocations()` pre-allocates space in `all_relocations_root` based on `rel_size / rel_entsz`, setting `all_relocations_count`. Then at line 1440, `dump_relocations()` is called which invokes `slurp_rela_relocs()` to parse relocation data. If the file is malformed and `slurp_rela_relocs()` fails (returning false at lines 592-593 or 622-623), `dump_relocations()` returns false at line 691 without populating the `r_symbol` fields in `all_relocations_root`. However, the pre-allocated `all_relocations_count` remains unchanged. Later, in `process_got_section_contents()` at line 1672-1673, the cleanup loop iterates through ALL `all_relocations_count` entries and calls `free(all_relocations_root[j].r_symbol)` - but for entries where `r_symbol` was never initialized (due to the earlier failure), this frees an uninitialized/invalid pointer, causing a use-after-free (CWE-763).

## Trigger Path

1. `bad.c:1682` `process_object(Filedata *filedata)` - Top-level entry point that orchestrates processing of an ELF file
   - Controls global state via `do_got_section_contents`, calls `process_relocs()` then `process_got_section_contents()`

   - Computes `rel_size / rel_entsz` at line 1438 and calls `update_all_relocations()` to pre-allocate global array with this count, then calls `dump_relocations()` at line 1440 with computed offset/size

3. `bad.c:670` `dump_relocations(Filedata *filedata, ...)` - Calls `slurp_rela_relocs()` which can fail
   - At line 690, calls `slurp_rela_relocs()`; if it returns false (due to malformed relocation data), returns false at line 691 without populating `r_symbol` fields in global array

4. `bad.c:576` `slurp_rela_relocs(Filedata*, uint64_t rel_offset, uint64_t rel_size, ...)` - Failure point
   - At lines 592-593 (32-bit) or 622-623 (64-bit), returns false when `get_data()` fails on malformed input, leaving allocation incomplete

5. `bad.c:1510` `process_got_section_contents(Filedata *filedata)` - Known vulnerable sink location  
   - At line 1672-1673, iterates ALL `all_relocations_count` entries (still set to pre-allocated value) and `free(all_relocations_root[j].r_symbol)` - frees uninitialized pointer for entries never populated due to failure at step 4
