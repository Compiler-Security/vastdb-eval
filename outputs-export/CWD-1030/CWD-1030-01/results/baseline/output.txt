

## Root Cause

The vulnerability is a **heap buffer under-read** due to incorrect array size calculation when zeroing memory. The `weights` array is allocated with `number_threads+1` elements at line 943, but `memset` at line 946 only zeroes `number_threads` elements (`number_threads*sizeof(*weights)` bytes instead of `(number_threads+1)*sizeof(*weights)`). This leaves `weights[number_threads]` (the last slot) containing uninitialized garbage values. When `DestroyBilateralTLS` is later called (at line 929), it attempts to free `weights[number_threads]` using `RelinquishMagickMemory`, passing an invalid/uninitialized pointer, leading to **CWE-763: Release of Invalid Pointer or Reference**.
