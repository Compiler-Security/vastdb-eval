

## Root Cause

The vulnerability is an out-of-bounds array access (CWE-129). The `substream` array in `struct eac3_info` (line 248) has only 1 element, but `hdr->substreamid` is read from 3 bits in the bitstream (line 593), allowing values 0-7. When this value is >=1, the sink at line 448 accesses `info->substream[hdr->substreamid]` out of bounds. No bounds check validates `hdr->substreamid` is within the array bounds before the array access.

## Trigger Path

1. `bad.c:400` `handle_eac3`
   - Entry point; parses AC-3 packet and writes to `info->substream[hdr->substreamid]` at line 448 using the unvalidated substreamid.

2. `bad.c:522` `avpriv_ac3_parse_header`
   - Called at line 412; parses bitstream and sets `hdr->substreamid = get_bits(gbc, 3)` at line 593 (3-bit value 0-7), returning control to `handle_eac3` without validation.
