## Root Cause

The vulnerability is an improper array index validation (CWE-129). The `substreamid` field is extracted from the bitstream using 3 bits (`get_bits(gbc, 3)` at line 593), allowing values 0-7. This value is then used to index the `substream` array at line 448 (`info->substream[hdr->substreamid].fscod = hdr->sr_code;`). However, the `substream` array is declared with only **1 element** (`struct { ... } substream[1];` at line 248). When `hdr->substreamid` is greater than 0 (e.g., 1-7), accessing `info->substream[hdr->substreamid]` causes an out-of-bounds array access, leading to memory corruption.

## Trigger Path

1. `bad.c:522` `avpriv_ac3_parse_header`
   - Parses the E-AC-3 audio frame header and sets `hdr->substreamid` from the bitstream (3 bits, value range 0-7)

2. `bad.c:400` `handle_eac3`
   - Calls `avpriv_ac3_parse_header` at line 412 to parse the header, then uses the returned `hdr->substreamid` to index the `substream` array at line 448 without bounds checking
