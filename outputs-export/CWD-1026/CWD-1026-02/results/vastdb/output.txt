## Root Cause

The vulnerability is CWE-416 Use-After-Free involving the `stored_frame` buffer in `SANMVideoContext`. The `av_freep()` macro at line 278 in `destroy_buffers()` frees `ctx->stored_frame` and sets the pointer to NULL. However, later in the same `decode_frame()` call, when processing the FTCH opcode (lines 587-590), the `process_ftch()` function at line 461 reads from `ctx->stored_frame` without checking if it's NULL. The bug is triggered when `init_buffers()` at line 286 fails (e.g., due to memory allocation failure) and calls `destroy_buffers()` at line 297, then execution continues in the same decode loop where FTCH is processed, leading to use-after-free.

2. `bad.c:730-731` `init_sizes` → `init_buffers` (inside decode_frame)
