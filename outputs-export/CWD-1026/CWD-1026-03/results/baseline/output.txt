## Root Cause

The vulnerability is in the `tr_variantWalk` function in `bad.cpp`. The stack (`struct SaveNode* stack`) is a dynamic array used for depth-first traversal. At line 660, a pointer `node` is obtained as `&stack[stackSize - 1]`. Later, when the stack needs to grow (lines 721-724 or 739-742), `tr_renew` is called which may reallocate the stack to a new memory address. If the reallocation succeeds, the old stack memory is freed and the `stack` variable gets the new pointer. However, `node` obtained earlier still points to the freed old stack buffer. At the sink location (line 668), `node->v` is accessed, which is a use-after-free since `node` now references freed memory.
