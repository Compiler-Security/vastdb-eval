

## Root Cause

The vulnerability is a use-after-free (CWE-416) caused by a dangling pointer in the `tr_variantWalk` function. The `stack` array is reallocated via `tr_renew` (lines 724, 742), but the local `node` pointer (set at line 660) still references the old memory location before reallocation completes. When the condition at line 668 (`tr_variantIsContainer(node->v)`) is evaluated on the next iteration, it accesses the freed `node->v`, triggering the use-after-free.

## Trigger Path

1. `bad.cpp:650` `tr_variantWalk`
   - The entry function that walks a variant tree using an internal stack array
   - At line 654: initializes stack with `tr_new(struct SaveNode, stack, stackAlloc)`
   - At line 660: sets `node = &stack[stackSize - 1]` to reference current stack element
   - At lines 714-728 (LIST) or 732-746 (DICT): when stack is full, calls `tr_renew` to reallocate stack to larger size
   - After reallocation, `stack` points to new memory but `node` still references old (freed) stack
   - At line 668: accesses `node->v` on dangling pointer, triggering the use-after-free

Since this is a standalone code snippet without a main() function, `tr_variantWalk` is the top-level entry point that would be called by external test code. The vulnerability triggers internally when processing nested LIST/DICT containers that cause stack reallocation.
