

## Root Cause

The vulnerability is a Use After Free in the MSL coder. In `WriteMSLImage` (line 1231), `msl_image` is a cloned Image pointer passed to `ProcessMSLScript` at line 1232. Inside `ProcessMSLScript`, at line 1284, the original `*image` pointer (referencing `msl_image` from the caller) is stored into `msl_info.image` via `MSLPushImage`. Later, at line 1325, `DestroyMSLInfo(&msl_info)` is called, which destroys all images in the `msl_info.image` array (line 1362), including the one that was cloned from the original `msl_image`. After `ProcessMSLScript` returns, the caller still holds `msl_image` at line 1233, but it now points to freed memory because `DestroyImageList` was called on it, causing a Use After Free when `DestroyImage(msl_image)` is called.

## Trigger Path

1. `bad.cpp:1231` `WriteMSLImage`
   - `msl_image` is assigned a cloned Image from the input parameter.

2. `bad.cpp:1232` `ProcessMSLScript`
   - Passes `&msl_image` to the function, where it becomes the `Image **image` parameter.

3. `bad.cpp:1284` `ProcessMSLScript` -> `MSLPushImage`
   - The original `*image` pointer is pushed into `msl_info.image`, making it subject to later destruction.

4. `bad.cpp:1325` `ProcessMSLScript` -> `DestroyMSLInfo`
   - Calls `DestroyImage` on images stored in `msl_info.image`, including the original pointer.

5. `bad.cpp:1233` `WriteMSLImage`
   - Attempts to call `DestroyImage(msl_image)` on the freed pointer, triggering the Use After Free.
