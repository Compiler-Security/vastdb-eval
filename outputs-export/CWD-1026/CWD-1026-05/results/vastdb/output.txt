## Root Cause

The vulnerability is a CWE-416 Use-After-Free. In `DetachBlob()` (bad.cpp:619), the `blob_info->data` is unmapped at line 629 via `UnmapBlob()`, which frees the mapped memory. The function then saves this now-freed pointer to a local variable `data` at line 640 and returns it (line 644). Back in the caller `ImagesToBlob()`, when it calls `RelinquishMagickMemory(blob)` at line 759, it frees the already-freed pointer returned by `DetachBlob`. Any subsequent use of `blob` after line 759 constitutes a use-after-free.

## Trigger Path

1. `bad.cpp:692` `ImagesToBlob`
   - Entry point function that processes image to blob format

2. `bad.cpp:755` `DetachBlob(images->blob)`
   - Calls DetachBlob to detach the blob from images structure

3. `bad.cpp:619` `DetachBlob(BlobInfo *blob_info)`
   - At line 629: `(void) UnmapBlob(blob_info->data,blob_info->length);` - unmaps/frees `blob_info->data`
   - At line 640-644: saves pointer to `data`, returns it to caller

4. `bad.cpp:759` `blob=RelinquishMagickMemory(blob);`
   - Frees the already-freed pointer from DetachBlob's return value

The vulnerable object is the `blob` pointer (originally `blob_info->data`). It becomes unsafe because `DetachBlob` returns a pointer to already-unmapped memory, which the caller then frees again with `RelinquishMagickMemory`. Any access to `blob` after line 759 triggers the use-after-free.
