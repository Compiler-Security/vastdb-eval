## Root Cause

The vulnerability is a **Use After Free** (CWE-416). In `DetachBlob()` at line 629, when a blob is memory-mapped, `UnmapBlob(blob_info->data, blob_info->length)` unmaps the underlying memory region via `munmap()`. However, the function still returns the original data pointer (saved at line 640) to the caller at line 644. Later, in `ImagesToBlob()` at line 759, this pointer is passed to `RelinquishMagickMemory(blob)`, which attempts to free memory that has already been unmapped by the kernel. The object becomes unsafe because the memory was released back to the OS but a dangling pointer to it is reused, leading to undefined behavior.

## Trigger Path

1. `bad.cpp:755` `ImagesToBlob`
   - Calls `blob=DetachBlob(images->blob)` to get the blob data pointer

2. `bad.cpp:644` `DetachBlob`
   - Returns the original data pointer after unmapping at line 629

3. `bad.cpp:629` `DetachBlob`
   - Calls `UnmapBlob(blob_info->data,blob_info->length)` to munmap the mapped memory

4. `bad.cpp:759` `ImagesToBlob`
   - Calls `blob=RelinquishMagickMemory(blob)` to free the already-unmapped memory (sink)
