## Root Cause

The vulnerability is a Use-After-Free in the `ff_rm_read_mdpr_codecdata` function. When parsing codec data with MIME type "logical-fileinfo", the code at line 640 calls `ff_free_stream(s, st)` which frees the `st` stream object. However, the code continues executing after the free without returning, eventually reaching the `skip` label (line 702) and returning from the function. The freed stream object remains in `s->streams` array, and if the caller or subsequent code tries to access it, a use-after-free occurs. The `mime` value passed from `rm_read_multi` (which reads it from the file at line 831) controls whether this free happens.

## Trigger Path

1. `bad.c:757` `rm_read_header`
   - Parses MDPR tag, reads `mime` from file at line 831 via `get_str8(pb, mime, sizeof(mime))`

2. `bad.c:843` `rm_read_header` → `rm_read_multi`
   - Calls `rm_read_multi(s, s->pb, st, mime)` when codec data starts with MLTI tag

3. `bad.c:715` `rm_read_multi`
   - Iterates streams and calls `ff_rm_read_mdpr_codecdata(s, s->pb, st2, st2->priv_data, size2, mime)` at line 748-749

4. `bad.c:638` `ff_rm_read_mdpr_codecdata`
   - The sink: `if(mime && !strcmp(mime, "logical-fileinfo"))` condition triggers `ff_free_stream(s, st)` at line 640, freeing the stream while the function continues execution
