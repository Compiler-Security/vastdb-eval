## Root Cause

The vulnerability is a NULL pointer dereference in the `raw_mantissa` array. The array `ctx->raw_mantissa` is allocated at line 756 as a pointer to `channels` pointers, and then each channel buffer is allocated at line 774 using `av_calloc(ctx->cur_frame_length, ...)`. However, this allocation only occurs when `sconf->floating` is true (line 751). When `sconf->floating` is false, both the outer array `ctx->raw_mantissa` (line 756) and all inner channel buffers (line 774) remain unallocated (NULL). At line 921, the code dereferences `raw_mantissa[c][i]` without checking if the allocation occurred, causing a NULL pointer dereference when processing non-floating point ALS streams.

## Trigger Path

1. `bad.c:629` `decode_init` - Initialize ALS decoder; allocates memory when `sconf->floating` is true at line 756 and 774
2. `bad.c:473` `read_frame_data` - Called from `decode_frame` at line 1021 to decode frame data; calls `read_diff_float_data` at line 618 when `sconf->floating` is true
3. `bad.c:812` `read_diff_float_data` - Decodes floating point data; dereferences `raw_mantissa[c][i]` at line 921 without checking if allocation occurred
