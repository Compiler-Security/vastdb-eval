## Root Cause

The vulnerability is a Null Pointer Dereference (CWE-476). The `pmt_iod` field in `GF_M2TS_Program` is set to NULL at lines 402, 408, and 417 in the `gf_m2ts_process_pmt` function under various conditions: when parsing a new IOD descriptor and deleting the old one before parsing succeeds, when the IOD has an incorrect tag, or when the IOD has empty ESDescriptors. Later, at line 1428 in `m2tsdmx_declare_pid`, the code dereferences `stream->program->pmt_iod->ESDescriptors` without checking if `pmt_iod` is NULL first. If the PMT processing set `pmt_iod` to NULL and the stream type is `GF_M2TS_SYSTEMS_MPEG4_PES` or `GF_M2TS_SYSTEMS_MPEG4_SECTIONS`, dereferencing this NULL pointer causes the crash.

## Trigger Path

1. `bad.c:342` `gf_m2ts_process_pmt`
   - Parses PMT section data, may set `pmt->program->pmt_iod = NULL` at lines 402, 408, or 417 when certain conditions are met (invalid IOD tag or empty ESDescriptors)

2. `bad.c:1016` `m2tsdmx_on_event` (calls `m2tsdmx_setup_program`)
   - Called when PMT section is found or updated (`GF_M2TS_EVT_PMT_FOUND`), passes program to setup function

3. `bad.c:1243` `m2tsdmx_setup_program`
   - Iterates program streams and calls `m2tsdmx_declare_pid` for each stream

4. `bad.c:1276` `m2tsdmx_declare_pid`
   - Processes each stream; at line 1428, dereferences `stream->program->pmt_iod->ESDescriptors` without NULL check when stream type is `GF_M2TS_SYSTEMS_MPEG4_PES` or `GF_M2TS_SYSTEMS_MPEG4_SECTIONS`, triggering the null pointer dereference
