## Root Cause

The vulnerability is a NULL pointer dereference in CWE-476. The `pmt_iod` field in the `GF_M2TS_Program` structure is set to NULL in function `gf_m2ts_process_pmt` (lines 402, 408, 417) when parsing the PMT section under certain conditions (empty IOD descriptor, wrong tag). Later, in function `m2tsdmx_declare_pid` at line 1428, the code dereferences `stream->program->pmt_iod->ESDescriptors` without checking if `pmt_iod` is NULL, causing a NULL pointer dereference.

## Trigger Path

1. `bad.c:1654` `m2tsdmx_initialize`
   - Entry point that initializes the demuxer and registers `m2tsdmx_on_event` as the callback handler (line 1661).

2. `bad.c:996` `m2tsdmx_on_event`

5. `bad.c:1276` `m2tsdmx_declare_pid`
