

## Root Cause

The vulnerability is a NULL pointer dereference in the IAMF audio demuxer. When parsing a scene-type audio element with `ambisonics_mode > 1`, the `ambisonics_config` function returns early at line 1780-1781 without allocating the `audio_element->layers` array. Later, in `iamf_read_header` at line 1891, the code accesses `audio_element->layers[0].substream_count` without checking if `layers` was initialized, causing a NULL pointer dereference when `ambisonics_mode > 1`.

## Trigger Path

1. `bad.c:1277` `audio_element_obu`
   - Parses audio element OBU and calls `ambisonics_config` for scene-type elements at line 1424.

2. `bad.c:1770` `ambisonics_config`
   - Reads `ambisonics_mode` from the bitstream. If `ambisonics_mode > 1` (line 1780), returns early without allocating `audio_element->layers`.

3. `bad.c:1854` `iamf_read_header`
   - Top-level entry point that processes parsed audio elements. Iterates over `iamf->audio_elements` and accesses the `layers` array at line 1891.

4. `bad.c:1891` (inside `iamf_read_header`)
   - Executes `audio_element->layers[0].substream_count` which dereferences a NULL pointer when `layers` was never allocated due to early return from `ambisonics_config`.
