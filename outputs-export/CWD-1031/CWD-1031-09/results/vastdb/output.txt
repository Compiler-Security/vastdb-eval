## Root Cause

The vulnerability is a NULL Pointer Dereference (CWE-476). In `ambisonics_config()`, when `ambisonics_mode > 1`, the function returns early at line 1780-1781 **without** allocating the `audio_element->layers` array (which is only allocated at lines 1792-1797). However, the `audio_element` object is still stored in the global context. Later, in `iamf_read_header()` at line 1891, the code accesses `audio_element->layers[0].substream_count` without checking if `layers` was allocated, causing a NULL pointer dereference when `layers` is NULL.

## Trigger Path

1. `bad.c:1854` `iamf_read_header`
   - Top-level entry function that processes IAMF audio format. Calls `ff_iamfdec_read_descriptors` at line 1860.

2. `bad.c:1708` `ff_iamfdec_read_descriptors`
   - Reads audio element descriptors from the stream. Calls `audio_element_obu` at line 1744.

3. `bad.c:1277` `audio_element_obu`
   - Parses audio element OBU. When audio element type is SCENE, calls `ambisonics_config` at line 1424. Stores the audio_element regardless of whether layers was allocated (line 1429).

4. `bad.c:1770` `ambisonics_config`
   - The vulnerable function. At line 1780, checks `if (ambisonics_mode > 1)` and returns early (line 1781) without allocating `layers`. The allocation at lines 1792-1797 is never reached.

5. `bad.c:1891` NULL pointer dereference
   - In `iamf_read_header`, accesses `audio_element->layers[0].substream_count` without checking if `layers` was allocated. When `ambisonics_mode > 1`, `layers` is NULL, triggering the dereference.
