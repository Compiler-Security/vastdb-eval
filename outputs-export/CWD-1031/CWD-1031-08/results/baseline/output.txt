## Root Cause

The vulnerability is in `class_object_index` at line 511 where `cl = rettv->vval.v_object->obj_class;` is executed. While the code checks if `rettv->vval.v_object` is NULL (lines 506-510), it does **not** check if the `obj_class` pointer is NULL. The `obj_class` field in the `object_S` struct is a pointer that can be NULL for incompletely initialized objects. When an object exists but has a NULL `obj_class` pointer, the code proceeds to line 517 where `cl->class_class_function_count` is accessed, causing a NULL pointer dereference.
