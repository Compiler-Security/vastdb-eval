## Root Cause

The vulnerability is a NULL Pointer Dereference (CWE-476) caused by a typo where the code accesses `sc->tts_data` instead of `sc->stts_data`. The `MOVStreamContext` struct contains two different fields: `MOVTimeToSample *tts_data` (line 270) which is **never** allocated, and `MOVStts *stts_data` (line 273) which is allocated in `mov_read_stts`. In `mov_read_trak` at lines 861 and 899, the code incorrectly accesses `sc->tts_data[i].duration`, but since `tts_data` was never allocated (remains NULL from the `av_mallocz` call at line 759), dereferencing it causes a NULL pointer dereference. The source operation at line 714 (`sc->stts_count = i;`) populates the correctly-named `stts_data` array, but the sink at lines 861/899 uses the wrong field name, leading to the vulnerability.
