## Root Cause

The vulnerability is a Null Pointer Dereference (CWE-476) in the MXF muxer code. In `mxf_parse_prores_frame()` at line 612, the `codec_ul` field is set to `NULL`, and may remain NULL if no matching PRORES profile is found in the lookup loop (lines 614-618). When `mxf_write_cdci_common()` is called to write the codec descriptor metadata at line 807, it dereferences `sc->codec_ul` via `avio_write(pb, *sc->codec_ul, 16)` without checking if it's NULL. If the header metadata is written before the first successful call to `mxf_parse_prores_frame()` (or if profile lookup fails), `sc->codec_ul` remains NULL, causing the null pointer dereference.

## Trigger Path

1. `bad.c:1204` `mxf_write_packet`
   - Top-level entry point called by the muxer's write callback for each packet; handles codec-specific parsing and header writing.

2. `bad.c:1233` `mxf_parse_prores_frame`
   - Called for AV_CODEC_ID_PRORES to look up the codec UL by profile. Sets `sc->codec_ul = NULL` at line 612, then attempts to set it based on profile match; if no match, it remains NULL.

3. `bad.c:852` `mxf_write_cdci_desc`
   - Called from the essence container descriptor table at line 1016 during header metadata writing, after `mxf_parse_prores_frame()` may have failed to set `codec_ul`.

4. `bad.c:854` `mxf_write_cdci_common`
   - Invokes `mxf_write_cdci_common()` which accesses `sc->codec_ul` at line 807.

5. `bad.c:807` (in `mxf_write_cdci_common`)
   - Calls `avio_write(pb, *sc->codec_ul, 16)` - the vulnerable dereference of potentially NULL pointer.
