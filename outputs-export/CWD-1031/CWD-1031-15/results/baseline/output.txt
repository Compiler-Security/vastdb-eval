

## Root Cause

The vulnerability is a null pointer dereference in `sc->codec_ul`. In `mxf_parse_prores_frame` (line 612), `sc->codec_ul` is explicitly set to NULL, and if no matching profile is found in the `mxf_prores_codec_uls` array, it remains NULL. Later, when `mxf_write_cdci_common` executes at line 807, it calls `avio_write(pb, *sc->codec_ul, 16)` without checking if `sc->codec_ul` is NULL, leading to dereferencing a null pointer.

## Trigger Path

1. `bad.c:1204` `mxf_write_packet`
   - Entry point for writing packets; calls `mxf_parse_prores_frame` at line 1233 when codec is PRORES

2. `bad.c:603` `mxf_parse_prores_frame`
   - Sets `sc->codec_ul = NULL` at line 612; if no matching profile found, leaves it NULL and returns 0 at line 621

3. `bad.c:852` `mxf_write_cdci_desc`
   - Called from `mxf_write_package` at line 1016 via function pointer during header metadata writing

4. `bad.c:628` `mxf_write_cdci_common`
   - Dereferences `sc->codec_ul` at line 807 via `avio_write(pb, *sc->codec_ul, 16)` without null check
