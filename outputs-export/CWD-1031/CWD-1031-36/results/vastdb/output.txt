## Root Cause

The vulnerability is a NULL Pointer Dereference (CWE-476) in the a.out/BFD reloc handling code. At line 2196 in `aout_32_swap_std_reloc_out`, the code dereferences `sym->section->output_section` without checking if `sym->section` is NULL. The symbol pointer stored at line 951 (`cache_ptr->sym_ptr_ptr = symbols + r_index`) can point to a symbol whose `section` field is NULL when an invalid extern symbol index is processed in the MOVE_ADDRESS macro. Specifically, in `aout_32_swap_std_reloc_in`, when `r_extern && r_index > symcount`, the code attempts to correct this by setting `r_index = N_ABS`, but if this correction doesn't properly validate the symbol or if the malformed symbol was already consumed earlier, the resulting `sym->section` pointer remains NULL.

## Trigger Path

1. `bad.c:2382` `copy_file`
   - This is the top-level function that reads/writes BFD files and calls the vulnerable code chain.

2. `bad.c:2372` `bfd_close` -> calls `i386linux_write_object_contents`
   - Closes the BFD and writes output, triggering the reloc writing.

3. `bad.c:2359` `i386linux_write_object_contents` -> calls `aout_32_squirt_out_relocs`
   - Writes object contents which iterates over relocations.

4. `bad.c:2263` `aout_32_squirt_out_relocs` -> calls `aout_32_swap_std_reloc_out` at line 2306
   - Iterates over relocations and calls swap_std_reloc_out for each.

5. `bad.c:2188` `aout_32_swap_std_reloc_out` (sink: line 2196)
   - **Vulnerable operation**: `asection *output_section = sym->section->output_section;`
   - This dereferences `sym->section` which can be NULL if the symbol stored from the source operation was invalid.

---

The source is in the read path, but VAST DB shows it's in a macro (MOVE_ADDRESS at line 948) expanded from `aout_32_swap_std_reloc_in` called during the earlier `slurp_reloc_table` phase. The flow is:
- `aout_32_slurp_reloc_table` -> `aout_32_swap_std_reloc_in` (MOVE_ADDRESS stores sym_ptr_ptr at line 951)
- Later: `aout_32_squirt_out_relocs` -> `aout_32_swap_std_reloc_out` (dereferences at line 2196)
