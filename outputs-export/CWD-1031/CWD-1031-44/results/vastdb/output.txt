## Root Cause

The vulnerability is a NULL Pointer Dereference (CWE-476). The `bit_read_TV` function at bad.c:2192 can return NULL in the error path at line 2202 when a buffer overflow is detected. When decoding TEXT entities in the version range R_2000 to R_2007, the code at line 2234 assigns this potentially NULL value to `_obj->text_value`. Later, when `output_TEXT` accesses `text->text_value` at line 1972 and passes it to `htmlescape`, it dereferences a NULL pointer, causing the crash.

## Trigger Path

1. `bad.c:3684` `main` (top-level entry)
   - Calls `output_SVG(&g_dwg)` to process the DWG file

2. `bad.c:2145` `output_SVG`
   - Iterates through DWG objects, calls `output_BLOCK_HEADER(ref->obj)` at line 2184 for each block

3. `bad.c:2076` `output_BLOCK_HEADER`
   - Iterates through owned entities, calls `output_object(obj)` at line 2126 in a loop

4. `bad.c:2006` `output_OBJECT`
   - Dispatches based on object type, calls `output_TEXT(obj)` at line 2026 when type is DWG_TYPE_TEXT

5. `bad.c:1958` `output_TEXT`
   - Dereferences `text->text_value` at line 1972 without NULL check
   - Calls `htmlescape(text->text_value, (int)dwg->header.codepage)` - **NULL pointer dereference occurs here**
