## Root Cause

In `PBMAC1_PBKDF2_HMAC` at bad.c:421, the code `pbkdf2_salt = pbkdf2_param->salt->value.octet_string;` dereferences `pbkdf2_param->salt` without checking if it is NULL. The `pbkdf2_param` is obtained from `PBMAC1_get1_pbkdf2_param(macalg)` which parses an ASN1 sequence. If the parsed sequence has a NULL or missing salt field, `pbkdf2_param->salt` will be NULL. When this NULL is dereferenced at line 421 to access `->value.octet_string`, it causes a NULL pointer dereference. The known vulnerable location at line 436 (`PKCS5_PBKDF2_HMAC(pass, passlen, pbkdf2_salt->data, ...)`) further dereferences `pbkdf2_salt->data`, but the crash occurs at line 421 when the NULL salt is first dereferenced.

## Trigger Path

1. `bad.c:583` `PKCS12_verify_mac`
   - Entry point that verifies MAC of a PKCS12 structure; calls `pkcs12_gen_mac`

2. `bad.c:517` `pkcs12_gen_mac` (via call at line 517/518)
   - When `pbmac1_md_nid != NID_undef`, calls `PBMAC1_PBKDF2_HMAC` passing `macalg` from parsed PKCS7 data

3. `bad.c:415` `PBMAC1_PBKDF2_HMAC` (call to `PBMAC1_get1_pbkdf2_param`)
   - Calls `PBMAC1_get1_pbkdf2_param(macalg)` to parse PBKDF2 parameters from ASN1; result stored in `pbkdf2_param`

4. `bad.c:421` `PBMAC1_PBKDF2_HMAC` (assignment)
   - Executes `pbkdf2_salt = pbkdf2_param->salt->value.octet_string;` - **NULL pointer dereference occurs here** when `pbkdf2_param->salt` is NULL (the salt field was missing or NULL in the parsed ASN1)
