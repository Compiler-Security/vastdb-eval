## Root Cause

The vulnerability is a NULL pointer dereference (CWE-476). At line 1676, `gf_m2ts_gather_section` is called with `NULL` as the third parameter (`ses`), which represents a section element stream. When the section processing completes in `gf_m2ts_section_complete` (line 1313), it calls `ts->on_event(ts, 24, &pck)` where `pck.stream` is set to `(GF_M2TS_ES *)ses` - which is NULL. Later, in `m2tsdmx_on_event` handler for event type `GF_M2TS_EVT_PAT_FOUND` (line 1196), the code dereferences `pck->stream->program->streams` without validating that `pck->stream` is non-NULL, causing the NULL pointer dereference.

## Trigger Path

1. `bad.c:1676` `gf_m2ts_process_packet`
   - Calls `gf_m2ts_gather_section(ts, ts->pat, NULL, &hdr, data, payload_size)` with `ses=NULL`

2. `bad.c:1583` `gf_m2ts_gather_section`
   - After processing section data, calls `gf_m2ts_section_complete(ts, sec, ses)` where `ses` is NULL

3. `bad.c:1313` `gf_m2ts_section_complete`
   - Sets `pck.stream = (GF_M2TS_ES *)ses` (which is NULL) and calls `ts->on_event(ts, 24, &pck)`

4. `bad.c:1196` `m2tsdmx_on_event`
   - Without NULL check, accesses `pck->stream->program->streams` where `pck->stream` is NULL, triggering the NULL pointer dereference
