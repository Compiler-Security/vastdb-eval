

## Root Cause

The vulnerability is a NULL pointer dereference at `bad.c:1318` in function `display_relocations()`. The function receives a `section` pointer parameter and directly accesses `section->sh_type` without checking for NULL. The root cause is that when `process_section_headers()` fails (line 1656), the code continues execution without properly preventing access to the NULL `section_headers` array. If `do_using_dynamic` is true, `do_reloc` is never set to false (line 1661-1662), so `process_relocs()` still executes and iterates over the NULL `section_headers` pointer, passing a NULL section to `display_relocations()`.

## Trigger Path

1. `bad.c:1613` `process_object`
   - Called as main processing entry for ELF file

2. `bad.c:1656` `process_section_headers`
   - Fails and returns false, but no early return; execution continues

3. `bad.c:1673` `process_relocs`
   - Called with `filedata` containing NULL `section_headers`

4. `bad.c:1568-1578` `process_relocs` (else branch)
   - Iterates over `filedata->section_headers` which is NULL
   - Calls `display_relocations(section, filedata, do_reloc)` with NULL section

5. `bad.c:1318` `display_relocations`
   - Directly accesses `section->sh_type` on NULL pointer, triggering CWE-476
