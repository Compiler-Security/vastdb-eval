## Root Cause

The vulnerability is a **NULL pointer dereference** in the `PID_IS_INPUT` macro. The macro at line 1405 is defined as `#define PID_IS_INPUT(__pid) ((__pid->pid==__pid) ? GF_FALSE : GF_TRUE)`, which dereferences the `pid` pointer to access its `pid` field without checking if `pid` itself is NULL. When `ctx->opid` is NULL (e.g., when `ctx->timescale` was never set), and `naludmx_set_dolby_vision()` is called from `naludmx_parse_nal_hevc()` (line 1646) upon receiving the first Dolby Vision NAL unit (DV_RPU), it invokes `gf_filter_pid_set_property(ctx->opid, ...)` at line 1503, passing a NULL `ctx->opid`. This eventually calls `gf_filter_pid_set_property_full()` at line 1407, where `PID_IS_INPUT(pid)` at line 1411 dereferences the NULL pointer, causing the null pointer dereference.

## Trigger Path

1. `bad.c:1688` `naludmx_process`
   - Called by the filter framework to process input packets; receives packets from `ctx->ipid`

2. `bad.c:519` `gf_filter_pid_get_packet` (via `ctx->ipid`)
   - Retrieves packet from input PID

3. `bad.c:1506` `naludmx_parse_nal_hevc`
   - Parses HEVC NAL units from the packet; when NAL type is `GF_HEVC_NALU_DV_RPU` and `ctx->nb_dv_rpu==1`, calls `naludmx_set_dolby_vision(ctx)`

4. `bad.c:1465` `naludmx_set_dolby_vision`
   - Called on first Dolby Vision NAL unit; calls `gf_filter_pid_set_property(ctx->opid, ...)` at line 1503 without checking if `ctx->opid` is NULL

5. `bad.c:1459` `gf_filter_pid_set_property`
   - Wrapper that calls `gf_filter_pid_set_property_full`

6. `bad.c:1407` `gf_filter_pid_set_property_full`
   - Calls `PID_IS_INPUT(pid)` at line 1411 to check if the PID is an input PID

7. `bad.c:1405` `PID_IS_INPUT` macro
   - Executes `__pid->pid` which dereferences a NULL `pid` pointer (when `ctx->opid` was never created due to zero `ctx->timescale`), causing the null pointer dereference vulnerability.
