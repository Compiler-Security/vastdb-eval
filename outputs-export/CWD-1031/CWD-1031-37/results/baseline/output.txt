

## Root Cause

The vulnerability is a **NULL pointer dereference** in the `coff_slurp_reloc_table` function. The `symbols` parameter, which is passed as `NULL` from `load_specific_debug_section` at line 985, flows through `bfd_canonicalize_reloc` → `coff_canonicalize_reloc` → `coff_slurp_reloc_table`. At line 759, when `dst.r_symndx` is valid and non-negative, the code computes `cache_ptr->sym_ptr_ptr = (symbols + obj_convert(abfd)[dst.r_symndx])`. Since `symbols` is `NULL`, this creates an invalid/fake pointer. Then at line 760, dereferencing `ptr = *(cache_ptr->sym_ptr_ptr)` causes the null pointer dereference.

## Trigger Path

1. `bad.c:985` `load_specific_debug_section`
   - Calls `bfd_canonicalize_reloc(abfd, sec, relocs, NULL)` with `symbols = NULL`.

2. `bad.c:827` `bfd_canonicalize_reloc`
   - Returns `-1` if format is not `bfd_object`; otherwise forwards to target-specific implementation via `BFD_SEND`.

3. `bad.c:799` `coff_canonicalize_reloc`
   - Calls `coff_slurp_reloc_table(abfd, section, symbols)` at line 815, passing the `NULL` symbols parameter.

4. `bad.c:707` `coff_slurp_reloc_table`
   - At line 759, computes `cache_ptr->sym_ptr_ptr = (symbols + obj_convert(abfd)[dst.r_symndx])` using the NULL `symbols` base, creating an invalid pointer.
   - At line 760, dereferences `ptr = *(cache_ptr->sym_ptr_ptr)`, triggering the null pointer dereference.
