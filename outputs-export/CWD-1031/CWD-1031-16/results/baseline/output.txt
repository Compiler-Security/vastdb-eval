

## Root Cause

The vulnerability is a NULL pointer dereference in `xml_sax_append_string` at line 154: `strlen(string)` is called without checking if `string` is NULL. The `string` argument is passed from `ent->value` in `gf_xml_sax_parse_intern`. When parsing XML entity definitions in `xml_sax_parse_entity`, an `XML_Entity` structure is allocated via `GF_SAFEALLOC` (line 199) and added to the entities list (line 215), but `ent->value` is only set when the closing quote separator is found (lines 217-222). If the XML input is malformed and the closing quote never appears, `ent->value` remains NULL. Later, when `gf_xml_sax_parse_intern` locates this entity and calls `xml_sax_append_string(parser, ent->value)` at line 541 with a NULL pointer, `strlen(NULL)` dereferences the NULL pointer, triggering CWE-476.

## Trigger Path

1. `bad.c:552` `gf_xml_sax_parse`
   - Exported API entry point, receives parser and string input, calls `gf_xml_sax_parse_intern`

2. `bad.c:576` `gf_xml_sax_parse_intern`
   - Parses XML content, locates entities via `gf_xml_locate_entity`, passes entity value to `xml_sax_append_string` at line 541

3. `bad.c:541` `gf_xml_sax_parse_intern`
   - Calls `xml_sax_append_string(parser, ent->value)` where `ent->value` can be NULL if entity definition was incomplete

4. `bad.c:154` `xml_sax_append_string`
   - Executes `strlen(string)` where `string` is NULL, causing the null pointer dereference
